# Путь Ушакова

## Описание проекта
 Путь Ушакова  интерактивная браузерная игра, в которой игрок проходит через ключевые моменты из жизни знаменитого русского адмирала Фёдора Ушакова. На каждой сцене предлагаются варианты решений, влияющие на ход истории, статистику персонажа и набор достижений. Игра работает полностью в клиенте, используя HTML, CSS и JavaScript, без серверной логики.

## Возможности
- Пошаговое прохождение с выбором вариантов решений.
- Динамический подсчёт баллов (отвага, вера, тактика и др.) и отображение их в финальном отчёте.
- Система достижений, сохраняемая в localStorage.
- Фоновая и сценическая музыка, включаемая/выключаемая в любой момент.
- Возможность делиться результатом через Web Share API или сохранить скриншот.

## Конфигурационные файлы
### data/config.json
```json
{
  totalScenes: 7,
  title: Путь Ушакова,
  startbg: files/images/start_bg.jpg,
  finalbg: files/images/final_bg.jpg
}
`
- 	totalScenes  количество сцен в игре.
- 	title  название, отображаемое в заголовке страницы.
-   startbg / inalbg  пути к фоновой картинке стартового и финального экранов.
```
### data/scenes.json
Содержит объект, где ключ  номер сцены, а значение  описание:
```json
1: {
  title: Перекресток судьбы,
  text: Год 1761...,
  fon: files/images/scene1.jpg,
  variants: [
    {text: Ехать в Петербург..., result: 1a, correct: true},
    {text: Остаться в Бурнаково..., result: 1b}
  ]
}
```
- fon  путь к фоновому изображению сцены.
- variants  массив вариантов с полями 	ext (текст кнопки), 
esult (идентификатор результата) и необязательным correct (правильный выбор).

### data/results.json
Отображает тексты последствий вариантов:
```json
1a: {
  title: Отправились в Питер,
  text: Вы покидаете родные берега...,
  fon: files/images/result1a.jpg,
  effects: [{name: courage, value: 2}],
  correct: true
}
```
- effects  массив, каждый элемент меняет счётчик (например, courage).
- correct  помечает вариант как правильный, учитывается при расчёте процента совпадения.

### data/sum.json
Набор правил, по которым формируется финальный текст в зависимости от достигнутых счётчиков и процента совпадения.

### data/achievements.json
Определения достижений и условий их получения (условия описываются в виде массива объектов с типом percent, counter или choice).

## Правила именования ресурсов
- **Изображения** находятся в ```files/images/``` и называются логично, например ```scene1.jpg```,  ```result1a.jpg```, ```start_bg.jpg```, ```final_bg.jpg```.

- **Музыка** находится в ```files/music/```:
  - Фоновая музыка  ```background_30.mp3```.
  - Звуки сцен  ```sceneX_50.wav```, где X  номер сцены.
- При добавлении новых сцен/результатов следует придерживаться той же схемы, чтобы движок автоматически находил файлы.

## Как создать свою игру на базе проекта
1. **Копировать структуру** ```data/scenes.json``` и ```data/results.json```, добавив новые сцены и варианты.
2. Добавить соответствующие изображения в ```files/images/``` и аудиофайлы в ```files/music/``` согласно правилам именования.
3. При необходимости изменить количество сцен в ```data/config.json``` (	totalScenes).
4. При желании поменять название игры  изменить поле 	title в ```config```.json.
5. При необходимости добавить новые счётчики или условия достижений  расширить sum.json и ```achievements.json```.

## Запуск локального сервера для тестирования
Проект работает как статический сайт, поэтому для корректной загрузки JSONфайлов необходимо запустить локальный HTTPсервер.
```bat
cd C:\Users\master\Documents\ML\game_arteck
python -m http.server 8000
```
После запуска откройте в браузере адрес:
```
http://localhost:8000
```
Игра загрузится, и вы сможете протестировать все функции. При работе через file://протокол загрузка JSON будет блокирована, о чём пишет консоль.
